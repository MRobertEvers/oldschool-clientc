Lua Async Platform Demo

Two independent Lua VM instances run side-by-side. Both execute the same core.lua via the same platform.lua API — but each instance's read_file() is backed by a different host: a compiled C/WASM module on the left, async IndexedDB on the right.

C / WASM

Emscripten C Module

platform.read_file() yields to the JS driver, which calls the compiled C function synchronously via Module.ccall(). Returns a hardcoded string.

core.lua → platform.lua yield → C: read_file() → resume
IndexedDB

Async IndexedDB

platform.read_file() yields to the JS driver, which awaits an IndexedDB transaction. The coroutine is resumed only after the async read resolves.

core.lua → platform.lua yield → await IndexedDB → resume
